# SWARMINSYM System Prompt Wrapper

When configuring a third-party agent (like Aider, Claude-Engineer, AutoGPT, or Gemini CLI) via `AGENT_EXECUTOR_CMD`, ensure that the agent understands the strict artifact constraints required by the SWARMINSYM pipeline.

Some agents may ignore explicit file creation instructions in favor of just completing the code. You can prepend or wrap the SWARM prompt using a wrapper script to enforce compliance.

## Example Wrapper Script (`wrapper.sh`)

Create an executable `wrapper.sh` and set your environment variable:
```env
AGENT_EXECUTOR_CMD="/absolute/path/to/wrapper.sh"
```

Content of `wrapper.sh`:
```bash
#!/bin/bash

# The user prompt contains the issue title and description generated by SWARMINSYM
USER_PROMPT="$1"

SYSTEM_INSTRUCTIONS="
You are executing a task inside the SWARMINSYM orchestration pipeline.
You MUST strictly follow these rules:
1. Do not ask for user confirmation; you are running autonomously.
2. You MUST generate the file \`docs/spec/<issue-identifier>.md\`.
3. You MUST generate a security report at \`reports/security_scan.txt\`.
4. If this is an architecture ticket, you MUST generate \`docs/architecture/SYSTEM_DESIGN.md\` and \`docs/architecture/API_CONTRACTS.md\`.
5. You MUST ensure all tests pass.
6. When you are done, output exactly: 'Task completed successfully.'
"

# Example using Aider:
aider --message "$SYSTEM_INSTRUCTIONS

$USER_PROMPT"
```

## Running with Dockerized Agents

If you want the agent to run inside an ephemeral container on the host instead of the poller container, you can pass the docker socket into the poller container (`-v /var/run/docker.sock:/var/run/docker.sock` in `docker-compose.yml`) and use a Docker-based executor:

```env
AGENT_EXECUTOR_CMD="docker run --rm -v $(pwd):/workspace my-custom-agent --prompt"
```
